# FX 利益最大化 調査・分析レポート

> 作成日: 2026-03-22
> 対象システム: 5戦略×32パラメータ IS/OOS 自動チューニング + ATR SL/TP + 上位TF EMA200確認

---

## エグゼクティブサマリー

現在のシステム（EMAクロス・ドンチャンブレイクアウト・トリプル確認・RSI×BB・夜間スカルパー）は IS/OOS 検証という正しい基盤を持っている。しかし「勝てるかどうか」は戦略選択だけでは決まらない。**トレードしてよい局面を絞り込む**・**リスクを正規化する**・**過剰最適化を定量的に検出する** という3軸の改善が最も費用対効果が高い。

---

## 1. プロのアルゴトレーダーが IS/OOS の先にやること

### ローリング・ウォークフォワード（多窓検証）

1回の 70/30 分割は「たまたまその期間に合っていただけ」の可能性がある。
プロはウィンドウをずらしながら繰り返し、**合成OOS損益曲線**（実際に生き残れたか）を確認する。

| Walk-Forward Efficiency (WFE) | 評価 |
|---|---|
| > 70% | 強い汎化性能 |
| 50〜70% | 許容範囲（軽微な最適化ブリード） |
| < 40% | 過剰最適化の疑い → 再設計 |

WFE = OOS年率リターン ÷ IS年率リターン

### 最低トレード数ルール

- IS期間: **100トレード以上**を必須とする
- OOS期間: **30トレード以上**なければ統計的に無意味
- 32パラメータ組み合わせのうち、この条件を満たさない組み合わせは **PFがいくら高くても除外**する

### コスト現実化

- スプレッドは固定ではなく、ボラティリティが上がると拡大する
- ドンチャンブレイクアウトのスリッページは想定の2〜4倍になりうる
- バックテストを **2× / 3× コスト** で再実行し、それでもプラスになるか確認する

## 2. プロフィットファクター（PF）を上げる実践的手法

### 目標値

| PF | 評価 |
|---|---|
| < 1.5 | ライブコストを考えると薄すぎる |
| 1.5〜2.0 | **実運用に適した範囲** |
| 2.0〜3.0 | 強いが OOS を必ず確認 |
| > 3.0（バックテスト） | ほぼ確実に過剰最適化 → 要調査 |

### 効果の大きい順に並べた改善手法

1. **セッション・ボラティリティフィルター**（→ 後述 §8）
   最も単純で最も効果が大きい。エントリーしてはいけない場面を排除するだけで PF が大幅改善する。

2. **上位TF トレンド確認の厳格化**（現在すでに実装済み）
   カウンタートレードは損失発生率がトレンド順張りの2〜3倍。逆張りを完全に排除することで PF が向上する。

3. **ATR 最小フィルター**
   ATR(14) が過去100本の X パーセンタイル未満のとき（レンジ圧縮期）はエントリーしない。SL/TP 比率が劣化するため。

4. **戦略ごとの RR 非対称最適化**
   - EMAクロス・トリプル確認: 1:2〜1:2.5（現状の 1:2 から少し上げてよい）
   - RSI×BB（平均回帰）: 1:1.5 に下げて勝率を上げる
   - ドンチャン: 後述のハイブリッドトレイリングストップを検討

5. **日次ドローダウン回路遮断器**
   口座の 1〜2% を超えたら当日は全戦略を停止。全戦略が同時に負けているのは異常局面の可能性大。

## 3. ローリング vs アンカード ウォークフォワード

| 方式 | IS ウィンドウ | 向いている戦略・時間足 |
|---|---|---|
| **ローリング（標準）** | 固定幅でスライド | H1/H4 の短期戦略全般。古い市場構造は不要 |
| **アンカード** | 開始固定・毎回拡大 | 日足/週足、夜間スカルパーのようにセッション構造が長期安定しているもの |

**現在のシステムへの推奨:**
- EMAクロス、ドンチャン、トリプル確認、RSI×BB → **ローリング**（IS:OOS = 4:1 = 12ヶ月:3ヶ月）
- 夜間スカルパー → **アンカード**（セッション効果は構造的に安定）

両方式の結果が近ければ、それはパラメータが本当にロバストであることの証拠になる。

## 4. レジーム検出による戦略切り替え（最大の改善ポイント）

### 2指標シンプルフレームワーク

| 指標 | 計算 | 閾値 |
|---|---|---|
| トレンド | SMA(50)[現在] - SMA(50)[5本前] | > 0 = 上昇トレンド |
| ボラティリティ | ATR(14) ÷ 終値 vs 過去100本MA | MA以下 = 低ボラ |

### 6つのレジームと対応する戦略

| レジーム | 推奨戦略 |
|---|---|
| 上昇 + 低ボラ | EMAクロス・ドンチャン・トリプル確認（最高の利益生成期） |
| 上昇 + 高ボラ | トレンド戦略のみ、ロット50%削減 |
| 下降 + 低ボラ | RSI×BB のみ（逆張り可能） |
| 下降 + 高ボラ | 夜間スカルパーのみ（アジア時間が穏やかなら） |
| 横ばい + 低ボラ | RSI×BB のみ |
| 横ばい + 高ボラ | **全戦略停止** |

### ADX による強化

```
ADX > 25 → 強トレンド → トレンド戦略エントリー許可
ADX 15〜25 → 発展途上 → ロット削減
ADX < 15 → レンジ/チョッピー → トレンド戦略禁止、RSI×BB のみ
```

現在のシステムの最大の弱点は、**チョッピーな横ばい局面でもエントリーしてしまうこと**。ADX < 15 フィルターを追加するだけで多くの負けトレードを排除できる。

### 発展：Hidden Markov Model（HMM）

Python の `hmmlearn` ライブラリで日次リターン系列を学習させ、「低ボラ安全期」と「高ボラ危険期」を自動識別。研究では 2006〜2023 年間でバイアンドホールドをアウトパフォームしている。

## 5. 戦略ポートフォリオ vs 単一最良戦略

**学術的コンセンサス:** 相関の低い複数戦略ポートフォリオは、単一最良戦略よりも**リスク調整後リターンが優れている**。

### 現在の5戦略の相関リスク

**本当の多様化ができていない組み合わせ:**
- EMAクロス・ドンチャン・トリプル確認は**上昇相場で全員同時ロング**になりやすい
- 3つの損益は高相関 → 実質1戦略と同じリスク

**本当の非相関源:**
- RSI×BB（平均回帰）← 貴重な真の多様化
- 夜間スカルパー（セッション構造）← 貴重な真の多様化

### 推奨改善

1. 各戦略ペアの直近60日損益相関を月次でモニタリング
2. 相関 > 0.7 になったペアは合算ロットを削減
3. RSI×BB と夜間スカルパーを優先確保 → 真の非相関アルファ源
4. 第6戦略として **XAU/USD（金）** の追加を検討（FX戦略と無相関になりやすい）

### 戦略ウェイト計算

等配分より、各戦略の**直近OOS損益曲線のボラティリティの逆数**で加重する。現在安定している戦略に多く、不安定な戦略に少なく配分する。

## 6. ATR ベースポジションサイジング（現システムの自然な次ステップ）

現在の問題: SL は 1.5×ATR で可変だが、**ロットが固定**。
→ ボラティリティが高い時にリスク過多、低い時にリスク過少になっている。

### ATR ポジションサイジング計算式

```python
atr_14 = calculate_atr(pair, 14)
dollar_risk_per_trade = account_equity * 0.01  # 1% リスク
stop_distance_pips = 1.5 * atr_14
pip_value = get_pip_value(pair)
position_size = dollar_risk_per_trade / (stop_distance_pips * pip_value)
```

### メリット

- ニュース・スパイク時に自動でロット縮小
- 閑散期に適切なロット増加
- 通貨ペア間の公平なリスク比較（USDJPY と GBPJPY のpip値は大きく異なる）
- 損益曲線が滑らかになる

### ケリー基準

| 方式 | 特徴 | 推奨 |
|---|---|---|
| フルケリー | 長期成長を最大化、ただしDD 50%超えも発生 | 使用禁止 |
| ハーフケリー | 最適成長の約75%、DD半減 | 経験者向け |
| クォーターケリー | 0.5〜1% リスク/トレード | **本番推奨** |

現在のシステムは ATR によって SL 幅が変わるため、固定ロットだとリスクが 0.3%〜3%+ まで乱高下している可能性がある。ATR サイジングでこれを修正するのが最優先。

## 7. FX で実際に機能する機械学習手法

### 何が過学習せずに汎化するか

**ランダムフォレスト（最も実践的）:**
- ブートストラップ + ランダム特徴選択で本質的に過学習耐性あり
- OOB（Out-of-Bag）誤差でホールドアウトなしのOOS推定が可能
- **特徴量重要度スコア**で「RSIは予測に貢献している、MACDはノイズ」と定量化できる
- `TimeSeriesSplit`（時系列分割）を使うこと。ランダムk-foldは禁止（データリーク）

**XGBoost:**
- L1/L2正則化で過学習を防ぐ
- 欠損値処理内蔵、指標間の相互作用を捉える
- FXの単体ML手法として最も実績が高い

**機能しないもの:**
- 10年未満の日足データでの深層ニューラルネットワーク（ほぼ確実に過学習）
- テクニカル確認なしのセンチメントのみモデル
- 月次以下の頻度でしか再学習しないモデル（陳腐化）

### 現システムへの具体的適用

1. 特徴量行列を作成: EMAスプレッド、ATR%、RSI値、MACDヒストグラム、BB位置、時刻、曜日
2. 目的変数: バイナリ（トレードが利益になったか）
3. IS期間で学習、OOS期間で検証（現在の70/30に対応）
4. **特徴量重要度で弱いシグナルを刈り取り**、既存インジをスリム化
5. モデルを**トレード品質ゲート**として使用: RF確信度 > 0.6 のときのみエントリー実行

## 8. 時間帯フィルター（統計的根拠あり）

### セッション別成績

| セッション | GMT時刻 | 日次pips範囲 | 最適通貨ペア | 対応戦略 |
|---|---|---|---|---|
| ロンドン | 08:00〜12:00 | 60〜100+ | EUR/USD, GBP/USD, EUR/JPY | EMAクロス・トリプル確認 |
| **ロンドン-NY重複** | **13:00〜17:00** | **80〜150** | **EUR/USD, GBP/USD** | **ドンチャン・トリプル確認** |
| NY | 13:00〜22:00 | 50〜100 | EUR/USD, USD/JPY | EMAクロス |
| 東京 | 00:00〜09:00 | 20〜40 | USD/JPY, AUD/JPY | 夜間スカルパー |

**重要な統計:**
- ロンドン-NY重複（13:00〜17:00 GMT）は**1日の取引量の37%**を占める（時間の19%に過ぎないのに）
- **火・水・木曜日**が最大の値動きを生む（月・金は弱い）
- 月曜朝のロンドン最初の2時間はギャップ埋めが支配的 → エントリー回避推奨
- 金曜17:00 GMT以降は週末ギャップリスク → 新規エントリー停止推奨

### 曜日フィルターの効果

```
月曜 08:00〜10:00 GMT: スキップ（ギャップ埋め）
金曜 17:00 GMT以降: 新規エントリーなし（週末リスク）
火〜木: 最優先エントリーウィンドウ
```

## 9. トレイリングストップ vs 固定TP（エビデンス）

**衝撃的な実験結果:**
- 通常のトレイリングストップ: 固定比率SL/TPより**利益22%減少**
- 改良版（上限付き条件付き）トレイリング: **15%減少**

> 「自動化されたトレイリングストップが固定決済機構より優れたシステムを見たことがない」
> — 定量的研究者による知見

### どちらを使うべきか

| 戦略 | 推奨決済方法 | 理由 |
|---|---|---|
| **EMAクロス** | 固定TP 1:2（現状維持） | ✅ |
| **ドンチャンブレイクアウト** | **ハイブリッド**を検証 | 大きな波を捕まえる戦略なので固定TPが利益を切る可能性あり |
| **トリプル確認** | 固定TP 1:2〜1:2.5 | 強トレンド時は 1:2.5 に拡大も |
| **RSI×BB** | 固定TP 1:1.5 | 平均回帰は早く完結する → 1:2より勝率重視 |
| **夜間スカルパー** | 固定TP（現状維持） | 静穏なアジア時間でトレイリングはエッジを殺す |

**ドンチャンのハイブリッドTP案:**
- ポジションの50%を 1:2 固定TPで決済
- 残り50%を 1×ATR のトレイリングストップで保有
- 「確実な部分」と「走る部分」を両立

## 10. 過剰最適化を避けながらパラメータを改善する7ステップ

### ステップ1: 最低サンプル数ゲート
IS 100トレード以上・OOS 30トレード以上を満たさないパラメータは除外。

### ステップ2: パラメータ安定性ヒートマップ
最適値の ±20% の範囲でPFを2D/3Dプロット。
- **ロバストな戦略**: 最適値周辺に**プラトー**（台地）が見える
- **過剰最適化**: 最適値に**急峰**（スパイク）が見える
- ルール: 最適パラメータの ±10% 範囲内で PF が 10〜15% 以内の低下に収まること

### ステップ3: OOS/IS 劣化比率
| OOS/IS PF 比 | 評価 |
|---|---|
| 0.7〜0.9 | 正常な劣化（本番可） |
| < 0.5 | 過剰最適化 |
| > 1.0 | ISが統計的に不運だった可能性（要確認） |

### ステップ4: マルチマーケット検証
各戦略を**3ペア以上**でテスト。EMAクロスが EUR/USD で機能するなら USD/JPY, AUD/USD でも正の期待値を持つはず。

### ステップ5: モンテカルロシミュレーション
OOS トレード系列に対して10,000回シミュレーション:
- 10%のトレードをランダムにスキップ（接続障害モデル）
- トレード順序をシャッフル（ラッキーシーケンスの排除）
- 価格に ±スリッページノイズを追加
- **P95ドローダウン（最悪ケース5%）がリスク許容範囲内か確認**
- **ルイン確率（-50% DDシミュレーション）= 0%** が本番基準

### ステップ6: ノイズテスト最適化
元データ + ±0.1%のランダムノイズを加えた50バリエーション、計51パスで同時最適化。51パス全体でスコアが高いパラメータのみが本物のロバスト解。

### ステップ7: パラメータ数の制限
**これが最も強力な過剰最適化防止策。**
各戦略の最適化ラウンドで追加するパラメータは 2〜3 個を上限とする。現在の 32 パラメータ組み合わせはすでに上限に近い。
