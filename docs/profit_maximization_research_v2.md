## A. レジーム検出：トレンドフォロー vs 平均回帰の切り替え

### ハースト指数（H）が最も信頼できるレジーム検出器

| H 値 | 市場レジーム | 使用すべき戦略 |
|---|---|---|
| H > 0.60 | 強トレンド | EMAクロス・ドンチャン・トリプル確認 |
| H 0.55〜0.60 | 中程度トレンド | トレンド戦略（リスク削減） |
| H 0.45〜0.55 | 不定/ランダム | ロット50%削減またはノートレード |
| H 0.40〜0.45 | 中程度平均回帰 | RSI×BB・夜間スカルパー |
| H < 0.40 | 強平均回帰 | 平均回帰のみ |

- 計算: ローリング100〜200本窓で R/S 分析
- Python: `hurst` パッケージ（pip install hurst）
- **注意**: バイアス補正後の中立閾値は 0.50 ではなく **0.55**

### 統計的ジャンプモデル（SJM）が HMM を上回る（2024年 Princeton/arXiv）

論文: *Shu, Yu, Mulvey (2024) — arXiv:2402.05272*

- 米国・ドイツ・日本の株価指数で 1990〜2023 年間テスト
- SJM 適用で年率リターンが **+1%〜4% 改善**（常時フルポジション比較）
- 従来 HMM の「レジームちらつき」を「ジャンプペナルティ」で解消
- 1本遅延（ライブ実行コスト）への耐性が高い

### 4つのレジームフレームワーク

1. **トレンド** → サイクル位相でタイミング + トレンド戦略
2. **平均回帰** → RSI×BB・夜間スカルパー
3. **チョッピー** → ロット削減、方向バイアスなし
4. **クライシス** → 資本保護のみ、自動取引停止

### レジーム確認の統計的検定

| 検定 | 判定基準 | 意味 |
|---|---|---|
| ADF 検定 | p < 0.05 | 平均回帰（定常）系列を確認 |
| 分散比検定 | 比 > 1 = トレンド、< 1 = 平均回帰 | トレンド vs 平均回帰 |
| ADX | > 25 と H > 0.55 の同時成立 | トレンド確認の最強コンビ |

**実装難易度: 中　期待改善: 誤シグナル 5〜12% 削減、年率 +1〜4%**

## B. ボラティリティベースのポジションサイジングとリスク管理

### ボラティリティターゲティング（VT）が多戦略システムに最適

Concretum Group の 40 先物市場横断比較:

| 手法 | 特徴 |
|---|---|
| **ボラティリティターゲティング (VT)** | 週次再調整で定常ボラ寄与を維持。最も滑らかな損益曲線・最高 Sharpe |
| ボラティリティパリティ (VP) | エントリー時固定、再調整なし。中庸 |
| VP + ピラミッディング (VP+P) | 2×ATR 毎に追加。大トレンドで +100bps だが急反転に脆弱 |

**5戦略システムには VT を推奨**（1戦略の外れ値が P&L 全体を歪めない）

```
ポジションサイズ = (口座資産 × 目標ボラ%) / (ATR × ATR倍率)
目標ボラ: 1トレード = 口座の 0.5〜1.0%
```

### ハーフケリー基準

- **フルケリー**: 長期成長最大化、ただし DD 50%+ 発生
- **ハーフケリー**: 成長の 75% を維持しながらボラを 25% 削減
- バフェット・ソロス・PIMCO 等が実際に採用
```
Kelly % = (勝率 × 平均利益 − 敗率 × 平均損失) / 平均利益
Half Kelly = Kelly % / 2
```

### 3段階ドローダウン回路遮断器（業界標準）

| ピークからの DD | アクション |
|---|---|
| −5% | リスク/トレードを 25% 削減 |
| −10% 〜 −15% | リスク/トレードを 50% 削減、最高グレードのみ |
| > −15% | 24〜72 時間取引停止、全システム点検 |

**日次・週次損失上限（Elder の 6% 月次ルールを週次化）:**
- 日次損失上限: 口座の 1.5〜3.0%
- 週次上限: 口座の 2〜3%
- 月次上限: 口座の 6〜8%

### 戦略間リスクパリティ

```
各戦略のウェイト = 1 / ローリング20本ボラ(戦略損益)
合計ウェイト = 1.0 に正規化（週次リバランス）
```

**実装難易度: 中　期待改善: DD 20〜35% 削減、Sharpe +0.2〜0.4**

## C. エントリータイミング最適化

### プルバックエントリー vs ブレイクアウトエントリー

| 市場/時間足 | ブレイクアウト勝率 | プルバック勝率 | 差 |
|---|---|---|---|
| FX H1〜H4 | 35〜50% | 50〜70% | **+10〜30pp** |
| 株式 日足 | 30〜45% | 50〜65% | +15〜20pp |
| 先物 日中 | 25〜40% | 40〜60% | +15〜20pp |

> **重要な注記**: ブレイクアウトは「当たった時の利益が大きい」。期待値ベースで比較が必要。ロンドン初時間帯の ORB（Opening Range Breakout）は 74.56% 勝率・PF 2.51 という実績あり。

**結論**: トレンド相場ではプルバックエントリー。真のブレイクアウト（数週間レンジ破り）はブレイクアウトエントリーが正当化される。

### 統計的根拠のあるローソク足パターン

QuantifiedStrategies が 75 パターンを S&P500 でバックテスト:
- **75 パターン中 66% がベンチマークをアウトパフォーム**
- ベアリッシュエングルフィング（買いで運用）: 勝率 **75.76%**、PF **2.73**（出来高確認付き）
- ブリッシュピアシングライン: 75 パターン中最高勝率

> **「どんな優秀なパターンも単独より、平凡なパターン + 良いフィルターの方が強い」**（出来高確認が最もインパクト大）

### セッションタイミングの統計的根拠

**ロンドン-NY 重複（UTC 13:00〜17:00）の確認:**
- EUR/USD・GBP/USD・USD/JPY で最も明確な流動性ゾーン
- ICT キルゾーン手法: ET 08:00〜11:00（主要エントリー）、ET 13:00〜16:00（副次）
- **この時間帯がスリッページ最小** → 指値注文の約定精度が最高

### 指値 vs 成行注文（FX）

| 状況 | 推奨 |
|---|---|
| プルバックエントリー（価格目標明確） | **指値注文** |
| ブレイクアウトモメンタム確認済み | **成行注文** |
| 経済指標発表中（NFP 等） | 成行（ただし 1〜3pip スリッページ覚悟） |

**実装難易度: 低〜中　期待改善: トレンド戦略で勝率 +8〜15%、シグナル頻度 15〜25% 減（質の向上）**

## D. エグジット戦略の最適化

### 部分利確の数学的真実

- 真のエッジを持つシステムでは: **単一エグジット（フルサイズ）が部分利確を上回る**
- 部分利確によりトレンドフォローの合計利益が **最大50% 削減**されるケースあり
- ただし: 損益曲線が滑らかになり、心理的ストレスと最大不利変動が削減される

**戦略別の推奨:**

| 戦略 | 推奨エグジット |
|---|---|
| EMAクロス・ドンチャン（トレンド） | 単一エグジット（2.0〜2.5×ATR TP）または1.5×ATR後にトレイル |
| RSI×BB・夜間スカルパー（平均回帰） | 50%を1×ATR T1で決済、残り50%はトレイル |

### トレイリングストップ性能データ

567,000 バックテスト設定の解析（KJTradingSystems）:
- トレイリングストップは多くの戦略で「わずかな改善」にとどまる
- ATR ベーストレイリング（最高値から 2×ATR）がパーセンテージ基準より FX では優秀
- **シャンデリアエグジット**（最高値から 3×ATR トレイル）: トレンド継続 vs ホイップソーのバランスが最良

### ブレイクイーブンストップ（損益分岐点ストップ）

エビデンスは相反するが:
- 移動後の期待値改善でなく損益曲線の滑らかさに寄与
- PF を **0.1〜0.3 単位 低下**させることが多い
- **有効なケース**: 強トレンド相場で 1.5R 達成後に SL を BE へ
- **逆効果なケース**: 平均回帰で通常の押し目が BE ストップをトリガー

### タイムベースエグジット（時間決済）

複数研究が一貫して確認:
- **システムのロバスト性を一貫して改善**（停滞ポジションから資金を解放）
- 「ランダムエントリーでも良いエグジットルールがあれば測定可能なリターン」
- 最適タイムストップ: **中央値勝ちトレード保有時間 × 2** で負けトレードを強制決済
- 平均回帰: 典型的な勝ちトレード決済時間の 2 倍以内に利益未達なら損切

**実装難易度: 中　期待改善: トレンド戦略で PF +0.2〜0.5**

## E. マルチタイムフレームシグナル確認

### 整合性が勝率を26.6ポイント改善（最大の発見）

8,734 トレードの実証研究（2023〜2024年）:

| 条件 | 勝率 |
|---|---|
| 上位TF + 下位TF シグナル整合 | **64.7%** |
| 上位TF 逆行シグナル | **38.1%** |
| **差** | **+26.6 ポイント** |

> これはコード変更なしで達成できる最大のシングル改善。現在の EMA200 チェックは正しい方向だが、より具体的なルールで強化できる。

### 最適タイムフレーム確認数

| TF 数 | 評価 |
|---|---|
| 1 TF | 不十分、誤シグナル多 |
| **2〜3 TF** | **実用上の最適点** |
| 4 TF 以上 | 過剰フィルタリング、トレード数が統計的有意性を下回る |

### 推奨マルチTF アライメントルール

| エントリーTF | トレンド確認TF | フィルターTF |
|---|---|---|
| M15 | H1 | H4 |
| **H1** | **H4** | **日足** |
| H4 | 日足 | 週足 |

パターン: TF N でトレード → TF 4N で確認 → TF 16N でフィルター

### EMA200 を超えた具体的確認ルール

現在の EMA200 確認に追加する高インパクト項目:

1. **EMA200 スロープ確認**（実装済みに近い）
   - EMA200[0] − EMA200[5] の符号がトレード方向と一致すること
2. **H4 RSI 50 フィルター**
   - ロング: H4 RSI > 50、ショート: H4 RSI < 50
   - RSI は別途計算不要（既存インジケータを再利用）
3. **HTF ローソク足構造**
   - 直近 3 本の HTF ローソクがトレード方向と逆の深いヒゲを持たないこと

**実装難易度: 低　期待改善: 勝率 +10〜20%、トレード数 15〜25% 削減**

## G. 機械学習によるシグナルフィルタリング（実践的）

### XGBoost が FX 分類タスクで最強

ScienceDirect 2025 論文（6 主要 FX ペア, 2000〜2023 年）の結果:
- XGBoost > Random Forest > Logistic Regression（FX 分類タスク）
- 最も予測力のある特徴量（実証済み）:

| 特徴量 | 計算方法 | 重要度 |
|---|---|---|
| RSI(14) | 1本前にラグ | ★★★ |
| ATR正規化リターン | `close変化 / ATR14`（1本前ラグ） | ★★★ |
| EMA200距離 | `(close − EMA200) / ATR14`（1本前ラグ） | ★★★ |
| ADX(14) | 1本前ラグ | ★★ |
| ローリングボラ | 20本リターンの標準偏差（1本前ラグ） | ★★ |
| MACD ヒストグラム | 1本前ラグ | ★★ |
| 相対出来高 | `volume / 20本平均出来高` | ★ |

### 先読みバイアス防止（3大落とし穴）

1. **特徴量計算バイアス**: 当日足 close で当日インジを計算 → **全特徴量を 1 本シフト（pandas: `df.shift(1)`）**
2. **分割バイアス**: `train_test_split(shuffle=True)` の使用 → **常に日付ベースの時系列分割**
3. **正規化バイアス**: 全データの平均・標準偏差で正規化 → **Scaler を訓練窓のみで fit、テスト窓は変換のみ**

```python
# 正しいウォークフォワード実装
for window in rolling_windows:
    X_train = features[train_start:train_end].shift(1).dropna()
    scaler = StandardScaler().fit(X_train)      # 訓練窓でのみ fit
    model = XGBClassifier().fit(scaler.transform(X_train), y_train)
    X_oos = scaler.transform(features[oos_start:oos_end].shift(1))  # 変換のみ
    predictions[oos_start:oos_end] = model.predict(X_oos)
```

### インクリメンタル学習 vs バッチ再学習

| 手法 | 特徴 |
|---|---|
| バッチ再学習（月次） | 推奨。長期エッジを保持 |
| インクリメンタル（週次） | XGBoost は `xgb_model` パラメータで対応。推論レイテンシが増加に注意 |
| AXGB（arXiv:2005.07353） | ミニバッチで新しい弱学習器を追加。コンセプトドリフトに最適 |

**実装難易度: 高　期待改善: シグナル精度（precision）+10〜20%**
**注意: 12ヶ月以上のライブデータが蓄積されてから構築**

## H. 取引システムの統計的検証

### 必要なトレード数（具体的な数値）

| 信頼度 | 最低トレード数 | 条件 |
|---|---|---|
| 基礎（CLT 適用） | 30 | 粗い基準のみ |
| 統計的有意の可能性 | **100** | Sharpe ≥ 1.0 で 95% 信頼 |
| 強い有意性 | **200** | 2〜3 完全マーケットサイクル |
| 機関投資家グレード | 500+ | 複数レジームにまたがる |

> **重要**: 「1つのレジームで500トレード < 強気・弱気相場をまたぐ150トレード」。**レジームの多様性がトレード数と同等に重要。**

### モンテカルロシミュレーション ベストプラクティス

- 最低 1,000 回シミュレーション（1,000 で大数の法則が機能）
- 手法: (1) リターンのランダムリサンプリング、(2) トレード順序のシャッフル、(3) ランダムスリッページ追加

**本番投入前に通過必須のゲート:**
- 95% のシミュレーションが黒字維持
- 最大シミュレーション DD < バックテスト DD の 2 倍
- 最小シミュレーション Sharpe > 0.5

### Bootstrap 信頼区間（PyBroker の BCa 法）

2025 年 arXiv 論文（arXiv:2510.12725）の非パラメトリックブートストラップ:
- 古典的ブートストラップはサンプル内では優秀だが **OOS では失敗**
- 非パラメトリック Bootstrap（40〜70パーセンタイル分位点）が滑らかで堅牢な結果
- **60パーセンタイル NPB 推定値を「現実的な期待成績」として使用**

### バックテスト過剰最適化の確率（Bailey & Borwein, CME Group）

複数パラメータをテストして最良を報告する場合の多重検定問題:
- N 個の組み合わせをテストして最良を報告 → 実効 p 値は N 倍
- **ボンフェローニ補正**: 最良のものに要求する p 値 < 0.05/N
- または **必要 Sharpe 閾値を引き上げる**（パラメータ数が多いほど高い Sharpe を要求）

**実装難易度: 中　効果: 直接的な利益改善ではなくリスク管理・デプロイゲート**
