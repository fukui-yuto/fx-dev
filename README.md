# FX Dashboard

MetaTrader 5 と連携した FX 取引分析プラットフォームです。
**自動チューニング**（ローリング3窓 IS/OOS 検証）で最良戦略を自動選択し、リアルタイムのエントリー・エグジットタイミングと具体的な Entry / SL / TP 価格をチャート上に表示します。

---

## 主な機能

| カテゴリ | 機能 |
|---|---|
| **自動チューニング** | ローリング3窓 IS/OOS 検証で 5 戦略×パラメータグリッドを自動最適化、24 時間キャッシュ |
| **多段フィルター** | ADX・ハースト指数・上位TF RSI50・セッション・プルバックの 5 段階で低品質シグナルを排除 |
| **ライブシグナル** | 最良戦略に基づくリアルタイム BUY/SELL/WAIT 判定、Entry / SL / TP 価格・推奨ロット（ボラティリティ調整済み）を表示 |
| **リアルタイムチャートマーカー** | ライブシグナル発生時に最新バーへ大型矢印（▲BUY / ▼SELL）を 1 秒ごとに更新表示 |
| **エントリー・エグジット表示** | バックテスト結果をチャートにマーカー表示（▲/▼ エントリー / ◎TP / ✕SL / ○シャンデリア / →EXIT） |
| **チャート** | 1 / 2 / 4 分割リアルタイム表示、30+ インジケーター、MTF 整合性パネル、エントリースコア、通貨強弱メーター |
| **バックテスト** | 40+ 戦略テンプレート、SL/TP 対応、20+ KPI 自動計算、損益曲線表示 |
| **ファンダメンタル** | USD/JPY 関連ニュース自動取得・翻訳・センチメント分析（5 分ごと更新） |
| **経済指標カレンダー** | ForexFactory 連携、今週・来週のイベント表示、強弱自動判定 |
| **データ管理** | MT5 からのローカル保存、SQLite 差分更新、保存済みデータ統計 |

## セットアップ

### 1. リポジトリをクローン

```bash
git clone https://github.com/your-username/fx.git
cd fx
```

### 2. 依存パッケージをインストール

```bash
pip install pipenv   # pipenv が未インストールの場合
pipenv install
```

### 3. 環境変数を設定（MT5 使用時のみ）

```bash
cp .env.example .env
```

`.env` を開いて MT5 のログイン情報を記入します。

```env
MT5_LOGIN=12345678
MT5_PASSWORD=your_password_here
MT5_SERVER=BrokerName-MT5
```

### 4. アプリを起動

```bash
pipenv run streamlit run app.py
```

ブラウザで `http://localhost:8501` を開くと起動します。

- **高インパクト指標カウントダウン** — 経済指標発表まで 15 分以内で警告を表示

## 自動チューニングの仕組み

### ローリング3窓 IS/OOS 検証

チャートを開いた際に自動実行されます（結果は 24 時間キャッシュ）。

1. 直近データを **IS（最適化期間）60%** と **3 つの OOS 窓**（13% / 14% / 13%）に分割
2. IS 期間で 5 戦略 × 各パラメータグリッドを全探索し、スコア上位 5 候補を選出
3. 3 つの OOS 窓で上位 5 候補を順番に検証し、**加重平均 OOS PF**（直近窓ほど重み大）が最高の戦略・パラメータを選択
4. 結果を `output/auto_tune_{symbol}_{timeframe}.json` にキャッシュ

```
データ全体
├── IS（最適化）     : 先頭 60%   グリッドサーチ実施
├── OOS 窓1（重み25%）: 60〜73%  汎化性能チェック
├── OOS 窓2（重み35%）: 73〜87%  汎化性能チェック
└── OOS 窓3（重み40%）: 87〜100% 直近パフォーマンス（最重要）
```

### 多段フィルターシステム

シグナルは以下のフィルターを順番に通過した場合のみエントリーとなります。

| フィルター | 内容 | 根拠 |
|---|---|---|
| **ADX ≥ 15** | レンジ相場（ADX < 15）ではエントリー禁止 | トレンド強度の最低基準 |
| **ハースト指数レジーム** | H < 0.45 ではトレンド戦略禁止、H > 0.55 では平均回帰戦略禁止 | ADX より精度の高いレジーム検出。誤シグナル 5〜12% 削減 |
| **上位TF EMA200 + RSI50** | 上位足の EMA200 方向・スロープ・RSI(14) > 50 がすべて一致すること | MTF 整合性で勝率 +26.6pp（8,734 トレード実証） |
| **セッションフィルター** | ロンドン / ロンドン-NY 重複時間帯のみ（戦略に応じて自動選択） | 流動性ゾーンでスリッページ最小化 |
| **プルバックフィルター** | EMAクロス・トリプル確認はエントリー足が EMA から 1.5×ATR 以内のみ | チェイスエントリー（値が離れすぎた追撃）を防止。勝率 +8〜15% |

### エグジット戦略

| エグジット種別 | 条件 | 対象戦略 |
|---|---|---|
| **TP（固定）** | エントリーから ATR × tp_mult pips で利確 | 全戦略 |
| **SL（固定）** | エントリーから ATR × 1.5 pips で損切 | 全戦略 |
| **シャンデリアエグジット** | 保有中の最高値（ロング）/ 最安値（ショート）から 3×ATR 引いたトレイリングストップ | EMAクロス・ドンチャン・トリプル確認（最適化で選択） |
| **タイムベースエグジット** | N バー経過後も TP 未達なら強制決済 | RSI×BB・夜間スカルパー（最適化で選択） |
| **逆シグナル決済** | 反対方向のシグナル発生時にドテン | 全戦略 |

### 自動チューニング対象戦略（5 種）

| 戦略名 | 種別 | 最適化パラメータ |
|---|---|---|
| EMAクロス | トレンド | 短期/長期 EMA 期間、RR 比、セッション、シャンデリア倍率、プルバック閾値 |
| ドンチャンブレイクアウト | トレンド | ブレイク期間、RR 比、セッション、シャンデリア倍率 |
| トリプル確認（EMA+RSI+MACD） | トレンド | EMA / RSI 期間、RR 比、セッション、シャンデリア倍率、プルバック閾値 |
| RSI×BB | 平均回帰 | RSI 閾値、BB σ、RR 比、タイムアウト本数 |
| 夜間スカルパー（4重確認） | スキャルプ | EMA / RSI / Stoch 期間、ATR 倍率、RR 比、タイムアウト本数 |

### ライブシグナルの信頼度判定

| 信頼度 | 条件 |
|---|---|
| 🔥 **high** | HTF 整合 ＋ OOS PF ≥ 1.2 ＋ シグナル発生 1 本前 |
| ⚡ **medium** | HTF 整合 ＋ OOS PF ≥ 1.0 |
| ❄️ **low** | 上記以外 |

- **HTF 整合性**: 上位足の EMA200 スロープ・価格位置・RSI(14) がすべてシグナル方向と一致
- **SL**: ATR × 1.5 を SL 幅として自動計算
- **TP**: SL × チューニング済み RR 比（1.5〜2.5）
- **推奨ロット**: ボラティリティターゲティング適用（現在 ATR / 平均 ATR の比でロットを自動調整。高ボラ時は縮小）

## 注意事項

- **MT5 は Windows 専用** です。macOS / Linux では MT5 連携は無効になりますが、内蔵サンプルデータでチャートやバックテストを利用できます。
- `output/`、`static/`、`data/ohlcv.db`、`.env` は `.gitignore` に含まれており、リポジトリには含まれません。
- バックテストおよび自動チューニングの結果はあくまで過去データに基づくシミュレーションです。将来の利益を保証するものではありません。
- 新しいフィルター（ハースト指数・プルバック・シャンデリアエグジット）は既存キャッシュには反映されていません。「再チューニング」ボタンを押すか 24 時間後に自動更新されます。
